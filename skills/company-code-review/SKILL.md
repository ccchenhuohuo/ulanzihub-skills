# 公司代码规范审查

按照优篮子公司的代码规范审查代码，确保代码质量。

## 适用场景

- 用户要求审查代码
- 用户要求检查代码质量
- 用户要求使用公司规范
- 代码提交前检查

## 审查要点

### 1. 命名规范

- 变量/函数使用小驼峰命名
- 类名使用大驼峰命名
- 常量使用全大写下划线
- 中文注释要准确表达意图

### 2. 代码结构

- 函数单一职责
- 适当提取公共逻辑
- 避免重复代码
- 保持代码简洁

### 3. 注释要求

- 公共 API 必须有注释
- 复杂逻辑必须解释
- 中文注释优先
- TODO 需要标注负责人

### 4. 错误处理

- 必须处理异常
- 错误信息要友好
- 关键操作需要日志

## 输出格式

审查完成后，输出以下格式：

```markdown
## 审查结果

### 通过 ✅
- [x] 命名规范
- [x] 代码结构

### 待改进 ⚠️
- [ ] 文件: src/utils.js:15 - 变量命名不够清晰

### 建议 💡
- 建议为函数添加 JSDoc 注释
```
